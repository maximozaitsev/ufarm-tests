ТЕСТИРУЕМЫЙ ПРОЕКТ: Ufarm

## 1. Общая информация
- **Название продукта**: Ufarm
- **Краткое описание**: Dapp для управления токенизированными финансами с разделением на модуль для инвестора (маркетплейс пулов) и модуль для управляющего фонда.
- **Тип системы**: Веб-приложение (dApp) с блокчейн-контрактами и набором сервисов (API).
- **Основные пользователи**: Инвесторы (маркетплейс) и управляющие фонда (фонд).

## 2. Архитектура и окружения
- **Основные компоненты**:
  - Модуль инвестора (маркет): `https://app.ufarm.digital/marketplace`
  - Модуль управляющего фонда (фонд): `https://fund.ufarm.digital/fund/`
  - Блокчейн-контракты пулов и токенов пулов
  - API-сервис `https://api.ufarm.digital`
- **Тестовые окружения**:
  - **DEMO** (Arbitrum test environment):
    - Маркет: `https://app.demo.ufarm.digital/marketplace`
    - Фонд: `https://fund.demo.ufarm.digital/fund/`
  - **PROD ARB** (Arbitrum main environment):
    - Маркет: `https://app.ufarm.digital/marketplace`
    - Фонд: `https://fund.ufarm.digital/fund/`
  - **PROD ETH** (Ethereum mainnet):
    - Маркет: `https://black.ufarm.digital/marketplace`
    - Фонд: `https://efund.ufarm.digital/fund/`
- **Доступы и роли**:
  - Инвестор — работает на стороне маркета.
  - Управляющий фонда — работает на стороне фонда (детализация ролей будет добавлена позже).
  - Есть тестовые адреса кошельков, для автотестов будут заведены отдельные адреса.

## 3. Ключевые бизнес-функции
- Маркет:
  - Депозит средств в пулы.
  - Вывод средств из пулов.
  - Просмотр портфолио инвестора (статистика, активные и прошлые пулы).
  - Управление инвестициями через UI маркета (выбор пула, переход в портфолио, управление кошельком).
- Фонд:
  - Создание и редактирование пулов внутри фонда.
  - Управление токенами инвесторов внутри пулов.
  - Операции с активами через arbitrary и другие dApp (своп, стейкинг и т.п.).

### 3.1 Маркетплейс (UI) — подключённый кошелёк

- **Главная страница маркета** (`/marketplace`):
  - Хедер:
    - Лого UFarm.
    - Табы: `All products` (selected), `My portfolio`.
    - Отображение текущей сети и кнопка подключения/отключения кошелька.
    - По нажатию на кошелёк открывается модалка кошелька с балансами и действиями (`FUND WALLET`, `SEND`, `DISCONNECT`).
  - Блок FAQ:
    - Кнопки/ссылки с вопросами (например, "How are funds secured?").
    - При клике открывается модальное окно с текстовым описанием.
  - Блок `All products`:
    - Список карточек пулов, отсортированных по убыванию `Value managed`.
    - По клику на карточку пула — переход на страницу пула `/marketplace/pool/{pool_address}`.

- **Страница портфолио** (`/marketplace/my-portfolio`):
  - Хедер: тот же, что на главной, но таб `My portfolio` выбран.
  - Блок `Overview`:
    - `My wallet` — баланс кошелька.
    - `My investments` — суммарный баланс инвестиций во всех пулах.
    - `All-time profit` — PnL (realized/unrealized).
  - Блок `My portfolio`:
    - Список карточек пулов, в которые есть или были инвестиции.
    - По клику на карточку — переход на страницу пула:
      - из портфолио — `/marketplace/my-portfolio/pool/{pool_address}`;
      - с главной — `/marketplace/pool/{pool_address}`.

- **Страница пула**:
  - В шапке блока пула (под основным хедером) расположены кнопки `DEPOSIT` и `WITHDRAW`; при скролле они "прилипают" к хедеру и остаются доступными.
  - Пул может принимать только USDC и/или USDT (конфигурируется управляющим фонда).
  - Депозиты совершаются в поддерживаемые стейблкоины (USDC/USDT), вывод также доступен только в эти токены.
  - Ниже на странице расположена история и связанные вкладки:
    - `My history` — депозиты и выводы текущего инвестора (адреса кошелька).
    - `My requests` — запросы на безгазовые депозиты и выводы.
    - `Transactions` — депозиты и выводы всех инвесторов.
    - `Actions` — изменения токенов в пуле (swaps и другие операции).

- **Модалка депозита**:
  - Выбор токена (USDC/USDT), отображение баланса.
  - Поле ввода суммы и кнопка `MAX`.
  - Переключатель `Gasless transaction` (вкл/выкл).
  - Кнопка подтверждения (например, `INSTANT DEPOSIT`).
  - Перед депозитом учитываются термы, проверки KYT/KYC (подробности будут уточняться отдельно).

- **Модалка вывода**:
  - Поле ввода количества пул-токенов и кнопка `MAX`.
  - Блок `You receive` с выбором стейблкоина для вывода (USDC/USDT).
  - Отображение курса обмена пул-токена на стейблкоин.
  - Кнопка `REQUEST WITHDRAWAL` для создания запроса на вывод.
  - Все выводы являются безгазовыми: запрос создаётся на стороне маркета и далее одобряется управляющим фонда.

- **Состояние без подключённого кошелька**:
  - Описанное выше поведение относится к состоянию **подключённого** кошелька.
  - При неподключённом кошельке:
    - Нет доступа к портфолио.
    - В хедере при нажатии на `Connect wallet` открывается модалка подключения кошелька (детали будут добавлены позже).

## 4. Особенности домена
- Пул — инвестиционный продукт, состоящий из набора токенов.
- Пул-токен — токен, создаваемый при создании пула; стартовая цена = 1.0. Цена пул-токена меняется в зависимости от стоимости токенов внутри пула.
- Управляющий фонда перераспределяет активы инвесторов по внешним dApp (свопы, стейкинг и т.п.), влияя на доходность пула.

## 5. Автотесты (ufarm-tests)
- **Типы тестов**:
  - `api` — API-тесты (без кошелька, проверяют данные и контракты).
  - `ui` — UI-тесты (Playwright, навигация и отображение).
  - `trx` — тесты транзакций (on-chain, депозит/вывод).
- **Наборы тестов (suites)**:
  - `smoke` — критичные happy-path проверки.
  - `regression` — стандартные регрессионные проверки, финансовые инварианты.
  - `extended` — глубокие и edge-case проверки.
- **Основной стек**:
  - Язык: Python 3.12+.
  - Фреймворк: `pytest` с `pytest.ini` для регистрации marks.
  - UI: `playwright` (Chromium, headless).
  - Отчёты: `allure-pytest` → HTML-отчёт на GitHub Pages с историей прогонов.
  - Модели данных: `pydantic` (валидация API-ответов).
- **Allure-разметка**:
  - `@allure.feature` / `@allure.story` / `@allure.title` — на английском.
  - `with allure.step(...)` — на русском, содержат конкретные значения для диагностики.
  - `@allure.severity` — CRITICAL (финансовые инварианты), NORMAL (структура).
  - `@allure.link` — ссылка на Swagger.
  - `allure.attach(JSON)` — тело ответа API прикреплено к каждому тесту.
  - `environment.properties` — окружение, URL, версия Python отображаются в отчёте.
- **CI/CD**:
  - GitHub Actions, запуск только вручную (`workflow_dispatch`).
  - Параметры: `test_type` (api/ui/trx/all), `test_suite` (smoke/regression/extended/all), `environment` (demo).
  - Secrets: `TEST_POOL_ID`, `TEST_WALLET_ADDRESS`.
  - Allure-отчёт публикуется на GitHub Pages, хранится 50 последних прогонов.
- **Тестовые данные**:
  - `TEST_POOL_ID` — UUID тестового пула (задаётся в `.env` или GitHub Secrets).
  - `TEST_WALLET_ADDRESS` — адрес тестового кошелька инвестора.
  - Все адреса передаются через конфиг, не хардкодятся в тестах.

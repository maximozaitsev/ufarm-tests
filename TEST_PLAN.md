Связанный контекст по системе: `TESTED_PROJECT_CONTEXT.md`

## 1. Цели автотестов
- **Краткосрочно**: проверить базовые happy-path сценарии инвестора в DEMO-окружении (депозит, вывод, отображение портфолио и истории).
- **Среднесрочно**: расширить покрытие UI и API для ключевых сценариев маркета и фонда.
- **Долгосрочно**: добавить проверку смарт-контрактов и ончейн-транзакций.

## 2. Маркет — UI-сценарии (первая очередь)

### 2.1 Депозит через All products
- Предусловия:
  - Подключён кошелёк с достаточным балансом USDC/USDT в DEMO.
  - Есть как минимум один пул, принимающий доступный стейблкоин.
- Шаги:
  1. Открыть `/marketplace` (DEMO).
  2. Убедиться, что таб `All products` активен и отображается список пулов.
  3. Выбрать пул из списка и перейти на страницу пула.
  4. Нажать `DEPOSIT` в шапке пула.
  5. В модалке депозита выбрать стейблкоин (если есть выбор), ввести сумму > 0 и < доступного баланса или нажать `MAX`.
  6. (При наличии) подтвердить термы/проверки, связанные с KYT/KYC (будет уточнено).
  7. Подтвердить операцию (например, `INSTANT DEPOSIT`) и завершить ончейн-транзакцию.
  8. Дождаться обновления состояния страницы пула (баланса/метрик).
  9. Перейти в `/marketplace/my-portfolio` и убедиться, что выбранный пул появился/обновился в `My portfolio`.
  10. Проверить, что в `My history` на странице пула появилась запись с типом `Deposit` и корректным значением.
- Ожидаемый результат:
  - Депозит успешно учтён в балансе пула и портфолио.
  - В истории инвестора есть корректная запись о депозите.

### 2.2 Вывод через My portfolio
- Предусловия:
  - Есть активная позиция инвестора в пуле (предварительно выполнен депозит).
- Шаги:
  1. Открыть `/marketplace/my-portfolio`.
  2. Найти пул с ненулевым балансом.
  3. Перейти на страницу пула из портфолио.
  4. Нажать `WITHDRAW` в шапке пула.
  5. В модалке вывода ввести количество пул-токенов или нажать `MAX`.
  6. Убедиться, что блок `You receive` показывает ожидаемый стейблкоин (USDC/USDT) и рассчитанную сумму.
  7. Нажать `REQUEST WITHDRAWAL`.
  8. Дождаться создания запроса на вывод (безгазовая транзакция).
  9. Проверить вкладки `My requests` и `My history` для текущего пула.
- Ожидаемый результат:
  - Вкладка `My requests` содержит запись о запросе на вывод.
  - В `My history` отображается операция вывода с корректным знаком и значением.

## 3. Маркет — API-сценарии (первая очередь, черновик)
- Получение списка пулов и проверка сортировки по `Value managed` по умолчанию.
- Получение детальной информации по пулу по адресу.
- Получение портфеля инвестора по адресу кошелька (структура и основные поля, без проверки точных значений доходности).

## 4. Ограничения для первых итераций
- Все тесты запускаются только против DEMO-окружения.
- Первые UI-тесты не валидируют точные числовые значения доходности, фокус на корректности навигации, появлении записей и базовой консистентности данных.
- Проверки, связанные с KYT/KYC, термами и полным ончейн-флоу, будут детализированы и добавлены позже.
